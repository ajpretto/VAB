| Solution | Target | Key Features | Gap for Anthony |

| **Built Technologies** | Lenders | $200B+ managed, AI Draw Agent | Wrong side of table (lender not GC) |
| **Procore Pay** | Enterprise | Part of Procore ecosystem | Requires full Procore buy-in |

| **Siteline** | Subs | Sub-focused pay apps, lien waivers | Wrong target (subs not GCs) |
| **Oracle Textura** | Enterprise | Full payment management | Enterprise scale/price |

**Key Insight from GCPay:**
> "55 minutes on average manually processing a single payment application"
> "A 4-6 day process to get a draw done has become a 4-6 hour process" — actual customer quote

**Strategic Positioning:**
- GCPay is the main competitor but targets larger GCs with ERP needs
- Built Technologies focuses on lender side (draw review), not GC side (draw creation)
- Siteline focuses on subcontractors, not GCs
- **Gap:** Smaller GCs who don't need full ERP integration but DO need draw automation
- **Anthony's advantage:** Inside access to actual GC workflow at Nicholas & Associates

**Recommendation:** 
Position as lightweight, focused draw package tool vs. enterprise platforms. Start with Nicholas as design partner (they get free tool, Anthony gets real feedback).
