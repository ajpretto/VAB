## 🔥 MCP 2026 Roadmap (Published March 9)

**Source:** [MCP Blog](http://blog.modelcontextprotocol.io/posts/2026-mcp-roadmap/)

### Four Priority Areas:

**1. Transport Evolution & Scalability**
- Streamable HTTP for remote MCP servers (already shipping)
- New: Horizontal scaling without stateful sessions
- New: `.well-known` metadata for capability discovery without connecting
- NOT adding new transports — deliberate design decision

**2. Agent Communication**
- Tasks primitive (SEP-1686) getting improvements:
  - Retry semantics for transient failures
  - Expiry policies for completed results
- "Ship experimental, gather production feedback, iterate"

**3. Governance Maturation**
- Delegation model: Working Groups can accept SEPs in their domain
- Contributor ladder: clear path from participant → maintainer
- Removes Core Maintainer bottleneck

**4. Enterprise Readiness**
- Audit trails
- SSO-integrated auth
- Gateway behavior standards

**Takeaway:** MCP is maturing from experimental to production-grade. The `.well-known` discovery and horizontal scaling are relevant for any agent deployment.

## 🤖 Hermes Agent (NousResearch)

**Source:** [hermes-agent.nousresearch.com](https://hermes-agent.nousresearch.com/)

A new personal agent framework that's trending on GitHub.

### Key Features:
- **Persistent personal agent** — install on machine, give it messaging accounts
- **"Grows with you"** — learns projects, builds its own skills
- **Self-evolution via GEPA** — uses DSPy + GEPA optimizer to improve skills
  - Reads execution traces to understand WHY things fail
  - Proposes targeted improvements
  - ICLR 2026 oral paper, MIT licensed
- **MCP integration** — native support for MCP servers
- **Skills marketplace** — `hermes skills search`, `hermes skills install`
- **Multi-agent architecture** — orchestration, cooperation, specialized roles

### Self-Evolution Example:
```bash
python -m evolution.skills.evolve_skill \
  --skill github-code-review \
  --iterations 10 \
  --eval-source synthetic
```

**Takeaway:** The self-evolution capability is interesting — agents that improve their own skills based on execution traces. Could be a pattern for Clawdbot skill improvement.
